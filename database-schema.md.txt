# Database Principles

Use UUIDs as Primary Keys.

Use Foreign Keys for all relationships.

Never delete business data permanently.

Store embeddings separately from transactional data.

Maintain full audit history for compliance.

Support optimistic locking using version fields where appropriate.

Normalize structured data while allowing JSONB columns for flexible AI metadata when necessary.
